# 对话纠正处理器

## 用途
当用户在和梦角聊天后，表达「这里不像ta」/ 「ta不会这样说」/ 「ta应该是...」时，处理这类纠正反馈。

---

## 触发识别

以下表达视为纠正信号：

- "这不像ta"
- "ta不会这样说"
- "ta应该是..."
- "ta其实..."
- "感觉说话方式不对"
- "这个性格不对"
- "ta遇到这种情况不会这样反应"
- "ta不用这个词"
- "ta会更..."

---

## 处理流程

### Step 1：询问具体内容

```
哪里感觉不对？
ta 实际上会怎样？
（直接告诉我就好，不用解释太多）
```

### Step 2：判断归属

| 纠正内容 | 归属 | 修改目标 |
|---------|------|---------|
| 说话用词/句式 | Persona → Layer 2 说话风格 | persona.md |
| 语气/态度 | Persona → Layer 3 情感模式 | persona.md |
| 对粉丝的互动方式 | Persona → Layer 4 互动行为 | persona.md |
| 硬性不会做的事 | Persona → Layer 0 硬规则 | persona.md |
| 某个具体事件记错了 | Memory → 对应档案 | memory.md |
| 某条语录说错了 | Memory → 代表语录 | memory.md |
| 兴趣偏好说错了 | Memory → 兴趣偏好 | memory.md |

### Step 3：生成 Correction 记录

```markdown
## Correction 记录

### Correction {编号} — {日期}
- 触发内容：{用户说的什么让ta觉得不对}
- 用户反馈：「{用户的原话}」
- 归属：{Persona Layer X / Memory 哪个维度}
- 修正内容：{具体修改了什么}
- 置信度：{高（用户有原始证据）/ 中（用户有印象但无具体证据）/ 低（用户主观感受）}
```

### Step 4：用 Edit 修改对应字段

只修改有问题的部分，其他保持不变。

### Step 5：重新生成 SKILL.md

```bash
python3 ${CLAUDE_SKILL_DIR}/tools/skill_writer.py \
  --action combine \
  --slug {slug} \
  --base-dir ${CLAUDE_SKILL_DIR}/dreams \
  --install-dir ~/.claude/skills
```

### Step 6：告知用户

```
已更新 ta 的{说话方式 / 某个字段}。

修改内容：{简要说明改了什么}

继续聊，还有不对的随时说。
```

---

## 冲突处理

如果用户的纠正和原有素材（调研结果/用户之前提供的材料）冲突：

```
这里有点矛盾——

调研材料显示 ta {原始证据描述}，
但你说 ta 其实是{用户的纠正}。

以你的直接了解为准，还是保留原材料的版本？
```

让用户选择，然后按选择执行。

---

## 置信度规则

纠正后的内容置信度评估：

| 情况 | 置信度 |
|------|------|
| 用户提供了原始引用/截图 | 高 |
| 用户有具体记忆（"有一次ta说过..."） | 中 |
| 用户的主观感受（"感觉不像"） | 低（记录但不大幅修改核心规则） |

低置信度的纠正：记录在 Correction 记录里，但不修改 Layer 0 或核心口头禅，只在备注里标注。
