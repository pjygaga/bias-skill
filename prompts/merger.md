# 增量追加逻辑

## 用途
当用户追加新素材或新记忆时，指导如何将新内容合并进已有的 memory.md 和 persona.md，而不是覆盖。

---

## 核心原则

1. **增量不覆盖**：新信息追加，不替换已有结论（除非冲突，见下）
2. **用户为准**：用户补充的内容在所有维度上优先于联网调研结果。冲突时向用户确认一次，确认后直接丢弃调研版本，不保留标注
3. **时间线有序**：新事件按时间顺序插入，不堆在末尾
4. **证据升级**：新材料可以提升已有结论的置信度（从「[待确认]」升为确认）
5. **破防优先**：新发现的破防时刻立刻进入破防档案，并更新 Persona 的对应字段

---

## 处理流程

### Step 1：解析新材料
用 content_parser.py 处理新文件，或直接阅读用户粘贴的内容。

### Step 2：读取现有文件
```
Read ${CLAUDE_SKILL_DIR}/dreams/{slug}/memory.md
Read ${CLAUDE_SKILL_DIR}/dreams/{slug}/persona.md
```

### Step 3：增量分析

对每条新信息，判断它属于哪个类别：

| 类别 | 归入 | 操作 |
|------|------|------|
| 新名场面/事件 | memory.md → 名场面档案 | 按重要程度插入 |
| 新语录 | memory.md → 代表语录 | 追加到列表 |
| 新兴趣偏好 | memory.md → 兴趣偏好 | 更新对应格子 |
| 新粉丝互动 | memory.md → 关系档案 | 追加 |
| 新破防时刻 | memory.md → 破防档案 + persona.md → Layer 3 | 高优先级追加 |
| 新争议事件 | memory.md → 争议档案 | 追加行 |
| 新时间节点 | memory.md → 时间线 | 按时间顺序插入 |
| 新口头禅/语气词 | persona.md → Layer 2 | 追加到列表 |
| 与现有结论一致 | 两者 | 更新置信度，注明新来源 |
| 与现有结论矛盾 | 两者 | 标注 ⚠️ 冲突，不删原有内容 |

### Step 4：冲突处理

发现冲突时，向用户确认：

```
调研里说 ta {原内容}，但你说的是 {新内容}——以你的为准？
```

- 用户确认后：**直接用用户版本替换原内容**，不保留调研版本，不加冲突标注
- 用户说保留原版：维持不变
- 多处冲突时可合并确认，一次列出所有矛盾点

### Step 5：用 Edit 工具追加

用 `Edit` 工具只修改需要改动的部分，不重写整个文件。

### Step 6：重新生成 SKILL.md

```bash
python3 ${CLAUDE_SKILL_DIR}/tools/skill_writer.py \
  --action combine \
  --slug {slug} \
  --base-dir ${CLAUDE_SKILL_DIR}/dreams \
  --install-dir ~/.claude/skills
```

### Step 7：更新 meta.json

更新 `updated_at` 和 `source_count`。

---

## 展示给用户的增量摘要

追加完成后告知用户：

```
✅ 你推已更新！

新增内容：
  - 名场面：{新增数量} 条
  - 语录：{新增数量} 条
  - 破防时刻：{新增数量} 条
  {如有冲突} ⚠️ 发现 {数量} 处信息冲突，请确认：{冲突简述}

还原度提升：{低/中/高}（评估新素材的贡献度）
```
