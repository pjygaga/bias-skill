## 它是什么

**bias-skill** 是一个 Claude Code Skill，能够把任何网络公众人物——偶像、主播、虚拟角色、游戏角色——蒸馏成一个你可以互动的高度定制化的 AI 角色。

它会自动联网调研目标人物的公开资料，结合你提供的私有素材（聊天记录、字幕、截图等），提取出两套核心数据：

| 组件 | 内容 |
|------|------|
| **Character Memory** | 人物时间线、名场面、代表语录、破防时刻等 8 个维度 |
| **Persona** | 说话风格、情感模式、价值观、肢体语言等 5 层画像 |

你不是只得到一段泛泛的人设介绍，而是可以立刻开始做这些事：

- 给自己设定专属称呼（你怎么叫 ta、ta 怎么叫你）
- 选择关系模式（初见 / 屏幕突破 / 平行宇宙）
- 定义你们的关系背景（邻居、同事、恋人、搭档等）
- 选择对话动作强度（纯对话 / 轻描写 / 沉浸模式）
- 随时追加证据素材并更新角色

最终生成的是一个独立、可直接对话、可持续迭代的 Skill 文件。

## 核心功能

### 自动调研

4 个 Agent 并行搜索公开资料：社媒语录、视频直播、粉丝评价、时间线动态。

### 证据驱动表达（区别于泛泛描述）

角色说话方式不是凭空编造，也不是模板化同人文风格，而是来自真实可追溯的网络证据：

- 公开采访 / 直播 / 社媒贴文 / 视频字幕
- 粉丝可交叉验证的高频语录与互动风格
- 你导入的私有素材（聊天记录、截图、口述等）

系统会基于证据抽取语气、词汇、节奏与情绪反应模式，尽量做到“有出处的像”，而不是“空泛的像”。

### 素材导入

支持多种格式的私有素材补充：

- 聊天记录（Bubble / 微信 / 私信截图或文本）
- 视频字幕（`.srt` / `.vtt` / `.txt` / `.json`）
- 图片识别（单张或整个文件夹批量扫描）
- B站弹幕（`.xml` 导出文件）
- 粉丝整理的语录合集
- 直接口述

### 关系预设

三种互动模式可选：

| 模式 | 说明 |
|------|------|
| **粉丝相遇** | ta 不认识你，第一次见面 |
| **屏幕突破** | ta 从屏幕里"走出来"，第一次和你说话 |
| **平行世界** | 自定义关系背景，ta 认识你 |

你可以在创建阶段直接指定：

- 你的专属称呼（如：宝宝、姐姐、队长、老师）
- ta 对你的称呼（如：你的小名、昵称、专属代号）
- 平行宇宙设定（时间线、身份关系、相遇方式、日常场景）

### 动作风格

`纯对话` / `轻描写`（默认） / `沉浸模式`（小说级场景描写）

### 持续进化

- 随时追加新素材，角色自动更新
- 对话中说「这里不像 ta」即可触发纠正
- 支持修改关系背景

### 多语言

中文 / 偶像母语 / 双语模式（原文 + 中文翻译）

## 快速开始

### 前提条件

- 已安装 [Claude Code](https://claude.com/claude-code)
- Python 3（用于字幕解析和 Skill 合并工具）

### 安装

将本项目克隆到本地后，在 Claude Code 中作为 Skill 目录加载即可：

```bash
git clone https://github.com/pjygaga/bias-skill.git
```

将 Skill 路径添加到 Claude Code 的 skills 配置中，或直接在项目目录下使用。

## 使用方式

### 创建新角色

```
/create-dream [角色名]
```

进入流程后会直接问你“关系与称呼”相关问题，你可以一步设定：

- 你怎么叫 ta
- ta 怎么叫你
- 你们是什么关系
- 平行宇宙里的背景设定

或用自然语言触发：

- `帮我蒸馏一个你推`
- `我想做一个XX的Skill`
- `给我做一个XX的角色`

### 与角色对话

创建完成后，使用生成的触发词：

```
/dream-{slug}
```

### 更新角色

```
/update-dream {slug}        # 追加新素材
```

或在对话中说：
- `这里不像 ta` — 触发纠正
- `修改关系背景` — 更改互动关系设定
- `换一个称呼叫我` — 调整专属称呼
- `我们换到另一个平行设定` — 切换关系与世界观

### 管理命令

```
/list-dreams                 # 查看所有已创建的角色
/delete-dream {slug}         # 删除指定角色
```

## 创建流程

```
Step 1  基础信息录入
        ↓ 公开名字 / 你的昵称 / ta 叫你什么 / 一句话描述 / 性格印象
Step 2  自动联网调研（4 Agent 并行）
        ↓ 社媒语录 / 视频直播 / 粉丝评价 / 时间线
Step 3  用户补充素材（可选）
        ↓ 聊天记录 / 字幕 / 截图 / 弹幕 / 语录 / 口述
Step 4  关系预设 + 动作风格选择
        ↓
Step 5  双线分析 → Character Memory + Persona
        ↓
Step 6  生成预览 → 用户确认
        ↓
Step 7  写入文件 → 安装 Skill → 可以聊了
```

## 已知问题与限制

### 调研相关

| 问题 | 说明 |
|------|------|
| **信息覆盖有限** | 调研基于公开可搜索的内容，付费内容、私域社区（如粉丝群、Discord 私频）无法触及 |
| **小众人物资料少** | 知名度较低的人物可能调研结果不足，建议多补充私有素材 |
| **时效性** | 调研为创建时的快照，不会自动追踪后续动态，需手动 `/update-dream` 更新 |

### 生成相关

| 问题 | 说明 |
|------|------|
| **公开人设偏差** | 角色基于公开形象生成，可能与私下性格存在差异；通过补充私有素材可缓解 |
| **破防时刻误判** | AI 可能错误判断哪些是"破防"（真实流露），需用户手动纠正 |
| **口头禅遗漏** | 基于高频词提取，可能遗漏情境性口头禅，建议在口述环节主动补充 |

### 使用相关

| 问题 | 说明 |
|------|------|
| **单人聚焦** | 目前仅支持单个人物蒸馏，不支持团体/多人关系动态 |
| **需要 Python 3** | 字幕解析和 Skill 合并依赖 Python，请确保环境中已安装 |
| **图片批量处理较慢** | 大量截图（>10 张）会分批处理，每批 5 张 |

### 素材冲突

当用户补充的素材与调研结果矛盾时，系统会确认一次后**以用户提供的为准**。这是设计如此——你比搜索引擎更了解你推。
